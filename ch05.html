Anatomy of a Shell Script

Every well-written script has 5 parts, in this order:

  1. The Shebang — #!/bin/bash — tells the OS which shell to use
  2. The Header — comments describing what, who, when, how to run
  3. Variables — settings defined at the top (paths, thresholds, emails)
  4. Functions — reusable blocks of code
  5. Main Logic — the actual work
A complete, properly structured script
#!/bin/bash
# ============================================================
# Script:  check_tablespace.sh
# Purpose: Check tablespace usage and alert if over 85%
# Author:  Imgard / Kudrant Academy
# Date:    2026-06-22
# Usage:   ./check_tablespace.sh
# ============================================================

# ---------- VARIABLES ----------
ORACLE_SID="PRODDB"
ORACLE_HOME="/u01/app/oracle/product/19c"
THRESHOLD=85
LOG_FILE="/home/oracle/logs/ts_check.log"
EMAIL="dba@company.com"

# ---------- FUNCTIONS ----------
log_msg() {
    echo "$(date '+%Y-%m-%d %H:%M:%S') - $1" >> $LOG_FILE
}

# ---------- MAIN LOGIC ----------
log_msg "Starting tablespace check"
# (your work goes here)
log_msg "Check complete"

exit 0
Follow this structure every time. Six months from now when you modify a script at 2am, you'll thank yourself for the clear header and top-level variables.
