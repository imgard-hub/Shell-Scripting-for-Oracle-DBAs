Scheduling with Cron

Cron runs your scripts automatically — daily, hourly, every 5 minutes, every Sunday at midnight. This is the final piece that makes automation real.

Cron syntax — 5 fields
# ┌───── minute (0-59)
# │ ┌─── hour (0-23)
# │ │ ┌─ day of month (1-31)
# │ │ │ ┌ month (1-12)
# │ │ │ │ ┌ day of week (0-7, 0/7=Sunday)
# │ │ │ │ │
# * * * * * command

  0  6 * * *   # Every day at 6:00 AM
 30  7 * * 1-5 # Weekdays at 7:30 AM
  0  * * * *   # Every hour
*/5  * * * *   # Every 5 minutes
  0  0 * * 0   # Every Sunday midnight
  0 22 * * *   # Every day 10:00 PM
A typical DBA crontab
# Edit: crontab -e   |   View: crontab -l

# Morning health check — 6:00 AM
0 6 * * * /home/oracle/scripts/daily_check.sh >> /home/oracle/logs/cron.log 2>&1

# Backup verification — 7:00 AM
0 7 * * * /home/oracle/scripts/backup_check.sh >> /home/oracle/logs/cron.log 2>&1

# Tablespace check — every 4 hours
0 */4 * * * /home/oracle/scripts/ts_check.sh >> /home/oracle/logs/cron.log 2>&1

# Alert log scan — every hour
0 * * * * /home/oracle/scripts/alert_scan.sh >> /home/oracle/logs/cron.log 2>&1

# Cleanup — Sunday at 2:00 AM
0 2 * * 0 /home/oracle/scripts/cleanup.sh >> /home/oracle/logs/cron.log 2>&1
Critical: Cron does NOT load your .bash_profile. Your script MUST set ORACLE_HOME, ORACLE_SID, and PATH explicitly. This is the #1 reason cron scripts fail.
Always redirect cron output: Add >> /path/log 2>&1 at the end. Without this, cron failures are invisible.
