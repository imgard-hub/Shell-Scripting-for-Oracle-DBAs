Variables — Storing Data

A variable is a name that holds a value. Instead of typing a long path ten times, store it once and reference it.

Defining variables
# IMPORTANT: NO spaces around the = sign!
NAME="Imgard"              # ✅ Correct
NAME = "Imgard"            # ❌ WRONG — causes error

# String variables
DB_NAME="PRODDB"
LOG_DIR="/home/oracle/logs"

# Numeric variables
THRESHOLD=85
PORT=1521

# Command output stored in a variable
TODAY=$(date '+%Y-%m-%d')
HOSTNAME=$(hostname)
DISK_USAGE=$(df -h / | tail -1 | awk '{print $5}')
Accessing variables
# Use $ to read a variable's value
echo "Database: $DB_NAME"
echo "Threshold: ${THRESHOLD}%"   # {} for clarity near other text

# ALWAYS quote your variables
echo "$DB_NAME"             # ✅ Handles spaces safely
echo $DB_NAME               # ⚠️ Breaks if value has spaces
The #1 beginner mistake: Putting spaces around =. NAME = "value" errors. NAME="value" is correct.
