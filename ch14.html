Connecting to Oracle from a Script

The here document (<<EOF) is how you embed SQL directly inside a shell script. This is the bridge between shell scripting and DBA work.

The pattern — memorize this
# Run SQL and display results
sqlplus -s / as sysdba <<EOF
SET LINESIZE 200 PAGESIZE 50
SELECT instance_name, status FROM v\$instance;
EXIT;
EOF
Critical: Inside a here document, escape the $ in Oracle views: v\$instance not v$instance. Without the backslash, Bash treats $instance as a shell variable.
Capture SQL output into a shell variable
DB_STATUS=$(sqlplus -s / as sysdba <<EOF
SET HEADING OFF FEEDBACK OFF
SELECT status FROM v\$instance;
EXIT;
EOF
)

if [ "$(echo $DB_STATUS | tr -d ' ')" = "OPEN" ]; then
    echo "✅ Database running"
fi
Run RMAN from a shell script
rman target / <<EOF >> $RMAN_LOG
BACKUP DATABASE PLUS ARCHIVELOG TAG 'NIGHTLY_FULL';
EXIT;
EOF

if [ $? -eq 0 ]; then
    echo "✅ Backup completed"
else
    echo "🚨 Backup FAILED"
    mail -s "BACKUP FAILED - $ORACLE_SID" $EMAIL < $RMAN_LOG
fi
