Planning Your Scripts

Before writing a single line, answer five questions:

  1. What problem does this solve? — one sentence
  2. What inputs does it need? — database name, paths, thresholds
  3. What output does it produce? — log file, email, screen
  4. What can go wrong? — database down, file missing, no space
  5. How will it run? — manually, via cron, triggered

Write your answers as comments in the script header. This takes 2 minutes and saves 30 minutes of rewriting.

Example: planning a backup check script
# PLAN:
# 1. PURPOSE: Verify RMAN backup completed for all databases
# 2. INPUTS:  List of SIDs, ORACLE_HOME path
# 3. OUTPUT:  Log file + email if any failed
# 4. ERRORS:  Can't connect → log and continue to next DB
# 5. SCHEDULE: Daily at 7:00 AM via cron
#
# PSEUDOCODE:
# For each database:
#   Connect, check v$rman_backup_job_details
#   If status != COMPLETED → add to alert list
# If alert list not empty → send email
# Write everything to daily log
Write pseudocode first. Then convert each line into real Bash. This is how professionals build scripts.
