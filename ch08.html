Operators

Operators compare values, do math, and test conditions. Bash uses different syntax than most languages — pay close attention.

TypeOperatorMeaningExample
Numeric-eqequal[ $A -eq $B ]
-nenot equal[ $A -ne $B ]
-gtgreater than[ $A -gt 85 ]
-gegreater/equal[ $A -ge 85 ]
-ltless than[ $A -lt 85 ]
-leless/equal[ $A -le 85 ]
String=strings equal[ "$A" = "OPEN" ]
!=not equal[ "$A" != "OPEN" ]
-zstring empty[ -z "$A" ]
File-ffile exists[ -f /path/file ]
-ddir exists[ -d /path/dir ]
-sexists & not empty[ -s /path/file ]
-xis executable[ -x /path/script ]
Logical&&AND[ $A -gt 0 ] && [ $B -gt 0 ]
||OR[ $A -gt 0 ] || [ $B -gt 0 ]
!NOT[ ! -f /path ]
Math$(( ))arithmeticRESULT=$((A + B))
DBA examples using operators
# Check if tablespace is over threshold
USAGE=90
if [ $USAGE -gt 85 ]; then
    echo "WARNING: Usage above 85%"
fi

# Check if alert log exists before reading
if [ -f "$ALERT_LOG" ]; then
    grep -i "ORA-" "$ALERT_LOG"
else
    echo "Alert log not found!"
fi

# Combine conditions: high usage AND production
if [ $USAGE -gt 85 ] && [ "$ENV" = "PROD" ]; then
    echo "CRITICAL: Production at ${USAGE}%!"
fi

# Calculate percentage
PCT=$(( (USED * 100) / TOTAL ))
echo "Usage: ${PCT}%"
Critical syntax: Spaces inside [ ] are REQUIRED. [$A -gt 85] fails. [ $A -gt 85 ] works.
