Best Practices & Common Mistakes

The 10 rules of DBA shell scripting:

  1. Always start with #!/bin/bash
  2. Always include a header comment — name, purpose, author, date
  3. Define all variables at the top — never hard-code in the middle
  4. Set Oracle environment explicitly — cron doesn't load your profile
  5. Always quote your variables — "$VAR" not $VAR
  6. Check $? after critical commands — don't assume success
  7. Log everything with timestamps — future you needs this
  8. Escape $ in Oracle views — v\$instance in here documents
  9. Test on non-production first — no exceptions
  10. Redirect cron output to a log

Common mistakes that break scripts:

  • Spaces around =: VAR="value" ✅ VAR = "value" ❌
  • Missing spaces in [ ]: [ $A -gt 5 ] ✅ [$A -gt 5] ❌
  • Using = for numbers: -eq for numbers, = for strings
  • Forgetting v\$ in heredocs
  • Forgetting chmod +x
  • Windows line endings — run dos2unix script.sh
The automation mindset: Automate one annoying task this week. Once you feel that time come back, you'll be hooked — and you'll never stop automating.
